Gemma Guide

Creative multimodal tool-use

Built for a Gemma hackathon, around the idea that an agent is only as capable as the tools you hand it. Gemma can reason about space but can't estimate depth, so I wired in Google's TIPS vision model as a tool: the agent "points" at an object with a bounding box, then pulls that object's depth straight from TIPS. Since TIPS also has a segmentation head, I could pinpoint the exact thing Gemma was pointing at, not just roughly where it was. The result is a guidance agent that understands where things are in a scene, not only what they are.

RoLLM (RoLlama3.2-1B)

A published model + low-level optimization

My first published model, now live under OpenLLM-Ro, the first open-source initiative for Romanian LLMs. It's a 4-bit, 1B Llama taught Romanian through a constrained continual-pretraining run, with both the model and the recipe built for efficient on-device inference. The recipe is an optimized QLoRA run on Unsloth, where I went deep on the levers that actually move a small model: sequence packing, LoRA rank, and data mix. The experiment I cared about most was whether higher educational-quality data makes a better model — and it did: language understanding improved across the board, while reasoning- and knowledge-heavy tasks gained the most from the filtered data. Getting there meant subclassing large parts of the Transformers library to shape the exact training loop I wanted, and profiling CUDA kernels with Nsight to keep it fast.

Hermes Curator

Production agent orchestration + safety

A system for reading, documenting, and evolving data from the web, built on the Hermes harness (an open-source agent framework aimed at local SLMs, with a lot of built-in hooks and guardrails). The principle is the interesting part: an orchestrator agent owns the high-level loop, anything potentially unsafe gets routed to tightly guarded subagents, and a dedicated subagent files everything into a persistent, interlinked markdown knowledge base using Hermes' llm-wiki skill (Karpathy's LLM Wiki pattern). That last bit means the curated knowledge compounds over time and stays cross-referenced, instead of being re-derived from scratch the way plain RAG would. The subagent split doubles as a security boundary and a context-engineering trick, keeping the main agent's context clean and focused. I built it out as a personal YouTube curator: a cron job scrapes my own recommendations and watch history with Playwright (YouTube offers no API for that personal data), the orchestrator documents and evolves the picks over time, and a Telegram gateway makes it easy to interact with day to day.

TuneMap

Knowledge graphs + adapter serving

Started as a way to explore my own music taste and ended up teaching me how agents can lean on knowledge graphs. It's a multi-agent setup: a main agent handles the conversation, and when a query needs translating into a graph query it calls a specialist "composer" agent — a model I fine-tuned for natural-language-to-Cypher and serve as a hot-swappable LoRA adapter in vLLM. So the system only spins up the specialist when it actually needs it, instead of paying for it on every call.

ChatTrace

Local, privacy-preserving RAG

My oldest project, and where I first learned to wire multiple models together with LangGraph. It's a RAG system over my own WhatsApp history, but the constraint that makes it interesting is that it runs entirely locally on a Mac through Ollama, so nothing private ever leaves the machine. It lets me trace and search old conversations while keeping the data fully private by design.

Claude Conversations

Dev tooling + cost-aware product thinking

A Claude Code plugin that scratches my own itch: picking up relevant conversations from previous days without scrolling forever. It uses smart compaction to generate a title and description for each past conversation, and I built it around keeping that summarization cheap and fast, so it stays a pleasant tool rather than a costly one. A small project, but a real exercise in balancing product quality against time and cost.
