TuneMap

# knowledge graphs + adapter serving

  • Knowledge Graphs
  • Multi-Agent Systems
  • vLLM
  • Fine-Tuning
> view on github

TuneMap turns my Apple Music library into a queryable knowledge graph. Ask a question in plain English and get a natural-language answer plus an interactive visualisation of the exact subgraph behind it. It started as a way to explore my own music taste and ended up teaching me how agents can lean on knowledge graphs.

It's a multi-agent setup: a main agent handles the conversation, and when a query needs translating into a graph query it calls a specialist “composer” agent, a model I fine-tuned for natural-language-to-Cypher and serve as a hot-swappable LoRA adapter in vLLM. So the system only spins up the specialist when it actually needs it, instead of paying for it on every call.

// what I learned

  • Agents can lean on knowledge graphs: translate natural language to Cypher with a specialist model instead of forcing one model to do everything.
  • Serving the fine-tuned composer as a hot-swappable LoRA adapter in vLLM means you only pay for the specialist when a query actually needs it.
  • Showing the exact subgraph behind an answer turns a black-box response into something you can inspect and trust.
