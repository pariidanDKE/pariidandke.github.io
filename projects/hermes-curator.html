Hermes Curator

# production agent orchestration + safety

> view on github

A system for reading, documenting, and evolving data from the web, built on the Hermes harness, an open-source agent framework aimed at local SLMs, with a lot of built-in hooks and guardrails. The principle is the interesting part: an orchestrator agent owns the high-level loop, anything potentially unsafe gets routed to tightly guarded subagents, and a dedicated subagent files everything into a persistent, interlinked markdown knowledge base using Hermes' llm-wiki skill (Karpathy's LLM Wiki pattern).

That last bit means the curated knowledge compounds over time and stays cross-referenced, instead of being re-derived from scratch the way plain RAG would. The subagent split doubles as a security boundary and a context-engineering trick, keeping the main agent's context clean and focused.

I built it out as a personal YouTube curator: a cron job scrapes my own recommendations and watch history with Playwright (YouTube offers no API for that personal data), the orchestrator documents and evolves the picks over time, and a Telegram gateway makes it easy to interact with day to day.

// what I learned

  • Splitting agents is both a security boundary and a context-engineering trick: route unsafe work to guarded subagents, keep the orchestrator's context clean.
  • An interlinked markdown knowledge base makes curated knowledge compound and stay cross-referenced, instead of being re-derived like plain RAG.
  • When there's no API for the data you need, a Playwright scraper on a cron is a pragmatic way in.
  • For an always-on agent feature, a cheap hosted API (like DeepSeek) does the work and frees up my own machine for other experiments.
